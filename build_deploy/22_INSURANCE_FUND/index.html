22 · PLATFORM PROTECTION MECHANISMS

Wellex deposit and yield protection architecture. Version: 5.0 | 05.03.2026 | Public

Overview​

Wellex uses a layered protection system to safeguard user deposits and yield distributions. Protection operates automatically — no user action required.

Protection Layers​

▸ Dynamic Yield Cap — when DeFi market conditions deteriorate, yield levels adjust proportionally rather than becoming unsustainable. Users always see real-time SR (Sustainability Ratio) on the dashboard.

▸ Liquidity Buffer — 5–15% of total platform value is held in immediately accessible form at all times, not deployed in DeFi strategies.

▸ Circuit Breaker — automated pause on deposits and withdrawals if abnormal platform activity is detected. Activated via multisig (3-of-5) with full user notification.

▸ Withdrawal Rate Limits — maximum 5% of platform TVL per hour / 15% per day / 40% per week. Protects against coordinated withdrawal events.

▸ Withdrawal Queue — FIFO queue activates automatically when limits are reached. Users see their position and estimated time in real-time.

▸ Smart Contract Coverage — dual audit (Trail of Bits + OpenZeppelin) + Bug Bounty ($250K on Immunefi) + Forta real-time monitoring.

▸ Stablecoin Diversification — USDC + USDT held simultaneously. Auto-rebalance triggers if either asset shows price deviation >1%.

Transparency Commitment​

▸ Real-time TVL visible on public dashboard ▸ SR (Sustainability Ratio) updated daily ▸ All yield sources disclosed (Aave, Compound, Curve, Ondo) ▸ On-chain verifiable payouts — every transaction has a public hash

User Rights​

▸ Non-custodial — Wellex never holds user private keys ▸ Withdrawal always available (subject to lock-up period per deposit tier) ▸ Private key export available anytime via Settings → Security ▸ Full transaction history exportable as CSV / PDF

All yield figures are targets — not guarantees. Actual results depend on DeFi market conditions and user WVI.

# 03 · WELLEX VITALITY INDEX (WVI)

> **Версия:** 6.0 | **Дата:** 12.03.2026 | **Статус:** Canonical
> **Категория:** Протокол
> **SSOT для:** формула WVI, биометрические компоненты, таблица уровней доходности
> **Зависимости:** 06_YIELD_PROTOCOL (доходность), 04_APP_UX (отображение), 25_TECH_REQUIREMENTS (спецификации)

---

## TL;DR

- WVI = ежедневный балл 0–100 из 3 биометрических компонентов: Эмоции (35%) + Восстановление (35%) + Физическая активность (30%)
- WVI определяет целевую доходность: **0–20%/месяц** (0% при WVI 0–39, до 20% при WVI 90–100)
- Период калибровки (День 1–7): WVI = 50 (нейтральный), доходность = 5%/мес фиксированная

## Назначение и область применения

Этот документ определяет методологию WVI — научное и алгоритмическое ядро Wellex. Охватывает формулу, компоненты, определение 11 эмоций, защиту от мошенничества, граничные случаи и каноническую таблицу WVI→Yield. Механика доходности: [06_YIELD_PROTOCOL](06_YIELD_PROTOCOL.md). Отображение в UX: [04_APP_UX](04_APP_UX.md) и [05_DASHBOARD](05_DASHBOARD.md).

## 1. Определение

**Wellex Vitality Index (WVI)** — проприетарный ежедневный балл здоровья (0–100), рассчитываемый на основе непрерывных биометрических данных Wellex Band. WVI — это «хэш-рейт» Emotional Mining: выше WVI → выше целевое распределение доходности.

**WVI в двухплатформенной архитектуре:**

| Платформа | Отображение WVI |
|-----------|----------------|
| **APP** | Полная детализация: WVI 0–100, 3 компонента, графики, тренды, AI-рекомендации |
| **WEB** | Виджет WVI + расчёт доходности в $, проектор дохода |

APP показывает только **%** доходности. WEB показывает полную финансовую детализацию.

---

## 2. Формула

```
WVI_daily = E_score × 0.35 + R_score × 0.35 + P_score × 0.30
WVI_monthly = Σ(WVI_daily) / N_days_in_month
```

Все субоценки нормализованы 0–100. Возрастная коррекция через скользящий 90-дневный персонализированный базис.
Дни без браслета: **WVI_daily = 0** (см. исключение Grace Days ниже).

**Grace Days:** 3 дня/месяц, когда WVI_daily = скользящее 7-дневное среднее активных дней (без штрафного нуля). Пользователь видит: «📅 Grace Day использован (осталось 2 из 3).»

**WVI Floor для стейкеров:** минимальный WVI = 25 для пользователей с активным стейкингом. Даже при нулевом ношении → распределение доходности = 0% (нет наказания, просто нет бонуса).

## 3. Компонент 1 — Эмоциональное состояние (35%)

Эмоциональный компонент — уникальный дифференциатор Wellex и ядро Emotional Mining.

**Ключевые метрики:**
▸ HRV (RMSSD, LF/HF ratio, pNN50)
▸ EDA — уровень кожной проводимости (SCL) и пики спонтанной реакции (SCR)
▸ Частота пульса в покое и её вариабельность

**Алгоритм определения:** многоклассовая LSTM + нейросеть с механизмом внимания. Вход: временной ряд HRV + EDA + акселерометр. Выход: распределение вероятностей по 11 эмоциям.

**11 эмоций — методология определения:**

| # | Эмоция | Паттерн HRV | Паттерн EDA | Уверенность |
|---|--------|-------------|-------------|:-----------:|
| 1 | Спокойствие | Высокий RMSSD, низкий LF/HF | Низкий SCL, редкий SCR | 95% |
| 2 | Радость | Умеренный RMSSD ↑, HF доминирует | Умеренный SCL ↑ | 88% |
| 3 | Возбуждение | Низкий RMSSD, высокий LF | Высокая частота SCR | 90% |
| 4 | Острый стресс | Очень низкий RMSSD, LF/HF > 4 | Всплеск SCL + частый SCR | 93% |
| 5 | Хронический стресс | Устойчиво низкий RMSSD (> 2ч) | Повышенный базовый SCL | 85% |
| 6 | Тревога | Низкий HF, нерегулярный RR | Повышенный SCL, спонтанный SCR | 82% |
| 7 | Фокус | Умеренный RMSSD, стабильный RR | Постепенный рост SCL | 80% |
| 8 | Усталость | Снижающийся тренд RMSSD | Сниженная EDA-реакция | 87% |
| 9 | Возбуждённость | Высокий ЧСС, низкий RMSSD, высокий LF | Всплески SCR > 10/мин | 85% |
| 10 | Расслабление | Высокий HF, RMSSD > 70мс | Снижающийся SCL, минимальный SCR | 92% |
| 11 | Медитативное состояние | Очень высокий HF, когерентный ритм | Очень низкий SCL, нет SCR | 90% |

**Логика Emotional Mining:** позитивные эмоции (Спокойствие, Радость, Фокус, Расслабление, Медитация) → высокий E_score → высокий WVI → более высокая доходность. Забота об эмоциональном состоянии буквально «майнит» экономическую ценность.

---

## 4. Компонент 2 — Восстановление (35%)

Отражает адаптивный резерв организма — способность восстанавливаться после физических и эмоциональных нагрузок.

**Ключевые метрики:**
▸ Качество сна: общая продолжительность, эффективность, фазы (лёгкий/глубокий/REM)
▸ Оценка восстановления: ЧСС в покое, HRV в первые 30 мин сна, температурный тренд
▸ Хронобиологический паттерн: регулярность цикла сон-бодрствование

Нормализация: результаты корректируются по индивидуальному скользящему 90-дневному базису и возрастной коррекции.

---

## 5. Компонент 3 — Физическая активность (30%)

Измеряет объём и качество ежедневной физической активности.

**Ключевые метрики:**
▸ Шаги, расстояние, калории (активные + базальные)
▸ MVPA (Физическая активность средней и высокой интенсивности) в минутах
▸ Интенсивность тренировок: зоны ЧСС, оценка VO₂max
▸ Баланс активность/восстановление (перетренированность = снижение P_score)

## 6. Таблица WVI → Целевая доходность (канон — SSOT)

> Это единственная каноническая таблица уровней доходности. Все документы обязаны ссылаться на эту версию.

| WVI (месяц) | Целевая доходность/мес | Реалистичный диапазон* | Годовой эквивалент |
|:-----------:|:---------------------:|:---------------------:|:-----------------:|
| 0–39 | 0% | 0% | 0% |
| 40–49 | 2% | 1–3% | ~24% |
| 50–59 | 5% | 3–7% | ~60% |
| 60–69 | 8% | 5–10% | ~96% |
| 70–79 | 12% | 8–15% | ~144% |
| 80–89 | 16% | 10–18% | ~192% |
| 90–100 | 20% | 12–20% | ~240% |

*Реалистичный диапазон зависит от условий DeFi-рынка, эффективности стратегий и SR-коэффициента.

> ⚠️ **Целевая доходность не является гарантией.** Фактическая доходность зависит от условий DeFi-рынка, WVI и распределения стратегий. Dynamic Yield Cap автоматически корректируется при SR < 1.0. Исторически средняя смешанная: **1.5–3.5%/месяц**.

**Период калибровки (Дни 1–7):** WVI = 50 (нейтральный). Доходность = **5%/мес фиксированная** (не 8%, не 20%). Честное позиционирование, без завышенных ожиданий.

**Wellness Boost (бонус за серию):**

| Серия | Бонус |
|:-----:|:-----:|
| 30 дней | +0.5% |
| 90 дней | +1.0% |
| 180 дней | +1.5% |
| 365 дней | +2.0% |

## 7. Модификатор распределения WVI

WVI определяет агрессивность распределения стратегий доходности (→ [06_YIELD_PROTOCOL](06_YIELD_PROTOCOL.md)):

| WVI | Распределение | Потолок доходности |
|:---:|--------------|:-----------------:|
| 0–39 | 80% DeFi+RWA · 20% Arbitrage | ~1%/месяц |
| 40–69 | 60% DeFi+RWA · 30% Arbitrage · 10% Prediction | ~4%/месяц |
| 70–89 | 40% DeFi+RWA · 30% Arbitrage · 30% Prediction | ~10%/месяц |
| 90–100 | 30% DeFi+RWA · 20% Arbitrage · 50% Prediction | ~20%/месяц |

Низкий WVI = консервативный портфель (защита капитала). Высокий WVI = агрессивное распределение (максимальная доходность).

## 8. Граничные случаи

| Ситуация | Обработка |
|----------|----------|
| Браслет не надет (день) | WVI_daily = 0 (или Grace Day при наличии) |
| Частичное ношение (< 6ч) | Пропорциональный расчёт по фактическому времени ношения |
| Двигательные артефакты | Фильтрация акселерометром, аномальные сегменты исключаются |
| Экстремальная тренировка | Разделяет усталость и перетренированность (перекрёстная валидация ЧСС + восстановление) |
| Болезнь / аномальные значения | Флаг «Recovery mode», увеличенный вес компонента восстановления |
| Смена часового пояса | Хронобиологическая коррекция (компенсация джетлага) |
| Ночная смена | Инверсия цикла сон-бодрствование — перекалибровка базиса |
| Нарушение сна (< 3ч) | R_score = 0, флаг уведомления |
| Первые 7 дней | Калибровка — WVI = 50, доходность = 5%/мес фиксированная |
| Grace Day (≤3/месяц) | WVI = скользящее 7-дневное среднее активных дней |

## 9. Система защиты от мошенничества

Целостность данных критична для доверия к доходности. Многоуровневая защита:

| Угроза | Контрмера |
|--------|----------|
| Имитация физического движения | Анализ когерентности HRV + акселерометр + гироскоп |
| Стороннее устройство | TEE-чип: криптографическая подпись каждого пакета данных |
| Модификация прошивки | Secure Boot, аттестация кода в Trusted Execution Environment |
| Replay-атаки | Временные метки + nonce в каждом пакете |
| Синтетические биосигналы | ML-детектор аномалий: живой человек vs. генератор сигналов |
| Передача устройства другому человеку | Биометрический отпечаток (уникальный базовый паттерн HRV) |
| Манипуляция со сном | Перекрёстная валидация: термометр + движение + фазы сна |
| Сетевые атаки | Сквозное шифрование BLE 5.3, мутационное тестирование API |

**Доверенный конвейер данных:**
```
Wellex Band (TEE подпись) → BLE E2E → Edge Server (проверка подписи) → Blockchain Oracle → WVI Engine → Расчёт доходности
```

**Fraud Score:** каждая сессия получает fraud_score (0–100). Превышение порога → автоматический сброс WVI за период + уведомление пользователя.

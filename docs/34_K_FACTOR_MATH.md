# 34 · K-FACTOR MATHEMATICS

## TL;DR

- K-factor = количество новых пользователей, которых приводит 1 существующий
- K > 1 = вирусный рост (экспоненциальный). K = 0.3–0.5 = здоровая добавка к платному трафику
- Wellex имеет 4 независимых viral loop с разными K. Суммарный K-target: 0.6–0.8 к M6
- Математика показывает: K 0.6 при CAC $30 → эффективный CAC снижается до $13.3

## 1. K-Factor: базовая математика

### Формула

```
K = i × c

где:
  i = количество приглашений от 1 пользователя за период
  c = конверсия приглашения в регистрацию (%)
```

### Что значит K-factor

| K-factor | Тип роста | Интерпретация |
|:--------:|-----------|-------------|
| K < 0 | Отток | Пользователи уходят быстрее, чем приходят |
| K = 0 | Линейный | Рост только через платный трафик |
| 0 < K < 1 | Суб-вирусный | Каждый пользователь приводит <1 нового. Добавка к CAC |
| K = 1 | Критическая масса | Каждый пользователь приводит 1 нового. Стабильное удвоение |
| K > 1 | Вирусный рост | Экспоненциальный рост без платного трафика |

### Влияние на CAC

```
Effective CAC = Paid CAC / (1 / (1 - K))

При K = 0.3: Effective CAC = $30 / 1.43 = $21.0
При K = 0.5: Effective CAC = $30 / 2.0 = $15.0
При K = 0.6: Effective CAC = $30 / 2.5 = $12.0
При K = 0.8: Effective CAC = $30 / 5.0 = $6.0
```

**При целевом K = 0.6 и Paid CAC $30 → Effective CAC = $12. Это 2.5× снижение стоимости привлечения.**

## 2. Четыре viral loop Wellex

### Loop 1: Referral Program (прямой реферал)

**Механика:** Пользователь приглашает друга → оба получают бонус.

| Параметр | Значение |
|----------|----------|
| Приглашений от 1 юзера (i) | 0.8/мес (avg, по базе beta-пользователей) |
| Конверсия приглашения (c) | 22% |
| **K этого loop** | **0.8 × 0.22 = 0.176** |

**Структура конверсии приглашения:**
```
100 приглашений отправлено
  → 65 открыли ссылку (65% open rate)
  → 34 дошли до registration page (52% CTR на ссылку)
  → 22 зарегистрировались (65% completion)
→ c = 22%
```

---

### Loop 2: WVI Social Share (органический вирус)

**Механика:** Пользователь делится WVI-карточкой в соцсетях → переходы → регистрации.

| Параметр | Значение |
|----------|----------|
| Доля юзеров, шарящих/мес (s) | 35% (target M3; сейчас ~20%) |
| Avg followers у шарящего | 800 |
| CTR на WVI-карточку | 1.2% |
| Конверсия перехода в регистрацию | 8% |
| **K этого loop** | **0.35 × (800 × 0.012 × 0.08) = 0.269** |

**Viral coefficient расчёт:**
```
1 пользователь шарит → 0.35 вероятность share за месяц
0.35 × 800 impressions = 280 impressions
280 × 1.2% CTR = 3.36 кликов
3.36 × 8% конверсия = 0.27 новых регистраций
K_loop2 = 0.27
```

**Что увеличивает K_loop2:**
- Streak NFT → более заметные карточки → CTR растёт до 2.0% → K = 0.44
- Leaderboard виральность («Команда #3 в MENA») → дополнительные share
- «Wellex Wrapped» (декабрь) → 1-дневный spike K до 2.0+

---

### Loop 3: MLM Partner Network (сетевой мультипликатор)

**Механика:** Партнёры рекрутируют пользователей в свои сети. Это структурный вирус.

| Параметр | Значение |
|----------|----------|
| Доля юзеров, ставших партнёрами (p) | 12% (target M3) |
| Avg рефералов от партнёра/мес | 4.2 |
| Конверсия в активного пользователя | 60% |
| **K этого loop** | **0.12 × 4.2 × 0.60 = 0.302** |

**Анализ партнёрской сети:**
```
1000 пользователей
  → 120 партнёров (ранг Explorer+)
  → 120 × 4.2 рефералов = 504 приглашения
  → 504 × 60% конверсия = 302 новых юзера
K_loop3 = 302 / 1000 = 0.302
```

**Компаундинг:** партнёры становятся пользователями → становятся партнёрами → рекрутируют. Каждый «ярус» добавляет ~15% от предыдущего эффекта.

---

### Loop 4: B2B Corporate (групповой вирус)

**Механика:** Компания покупает Wellex for Teams → 50–500 сотрудников онбордятся одновременно → часть становится партнёрами или шарит в соцсетях.

| Параметр | Значение |
|----------|----------|
| B2B клиентов в месяц (Phase 2) | 3 |
| Avg сотрудников на B2B клиента | 120 |
| Доля ставших partner/referral | 15% |
| Avg рефералов от B2B-пришедшего | 1.8 |
| **K этого loop** | **3 × 120 × 0.15 × 1.8 / 1000 users = 0.097** |

Небольшой K — но каждый B2B клиент = 120 пользователей сразу, что ускоряет базу для loops 1–3.

## 3. Суммарный K-factor

**Важно:** loops не суммируются линейно — они частично пересекаются. Реальный суммарный K:

```
K_total = K1 + K2 + K3 + K4 - overlap(~15%)

Phase 1 (M1–M3):
K_total = 0.176 + 0.27 + 0.302 + 0.097 - 0.08 = 0.765
```

Но реалистично — в первые месяцы loops работают не на полную мощность:

| Период | K реальный | K target |
|--------|:----------:|:--------:|
| M1 | 0.20 | 0.25 |
| M3 | 0.35 | 0.40 |
| M6 | 0.55 | 0.60 |
| M12 | 0.65 | 0.70 |

## 4. Growth modeling — как K влияет на рост

### Базовая модель (M1: 1000 paid users, monthly paid acquisition: 500)

```
M1: 1000 (paid) + 1000 × K = base
M2: (M1 × (1-churn) + 500 paid) × (1 + K)
...
```

**Сценарии при разном K (10 месяцев, churn 8%/мес, 500 paid/мес):**

| Месяц | K=0.2 | K=0.4 | K=0.6 | K=0.8 |
|:-----:|:-----:|:-----:|:-----:|:-----:|
| M1 | 1,100 | 1,200 | 1,300 | 1,400 |
| M3 | 2,800 | 3,600 | 4,600 | 6,000 |
| M6 | 5,200 | 8,100 | 13,500 | 23,400 |
| M10 | 7,800 | 15,200 | 31,200 | 71,000 |

**Вывод:** разница между K=0.4 и K=0.6 к M10 = **31,200 vs 15,200** — в два раза. Это разница между $6.2M ARR и $3.1M ARR.

## 5. Referral Economics

### Unit Economics реферала

| Параметр | Значение |
|----------|----------|
| Стоимость реферального бонуса | $5 + 10% M1 subscription = $5 + $1.9 = $6.9 USDC |
| CAC через реферал | $6.9 |
| LTV реферала (avg) | $19 × 18 мес × (1-15% churn) × 85% margin = $247 |
| **LTV / CAC** | **247 / 6.9 = 35.8×** |

Для сравнения: платный трафик Meta Ads → CAC $28, LTV/CAC = 247/28 = **8.8×**

**Реферальный канал в 4× эффективнее платного трафика по LTV/CAC.**

---

### Оптимизация реферального CAC

**Где теряется конверсия (и что с этим делать):**

```
100 приглашений
  → 65 открыли (-35% не открыли) 
     FIX: персонализация ссылки с именем реферера
  → 34 прошли на registration page (-48% закрыли)
     FIX: landing page A/B test (сейчас 52% CTR → target 65%)
  → 22 зарегистрировались (-35% не завершили)
     FIX: упрощённый onboarding (email → 1 click)
  → 18 активировали браслет (-18% не активировали)
     FIX: onboarding video + SMS reminder
→ Итого: 18% effective conversion (target: 25%)
```

**При улучшении конверсии с 22% до 30% (через UI/copy оптимизацию):**
- K_loop1 вырастет с 0.176 до 0.240
- Суммарный K растёт с 0.55 до 0.62
- При 10K users — дополнительно +700 users/мес бесплатно

## 6. K-factor мониторинг

### Метрики для ежемесячного трекинга

| Метрика | Формула | Где мерить |
|---------|---------|-----------|
| **Viral K** | (New users from referrals) / (Total users prev month) | Analytics dashboard |
| **Invite Rate** | (Users who sent ≥1 invite) / (Total active users) | Events tracking |
| **Invite Conversion** | (Registrations from invites) / (Total invites sent) | Referral system |
| **Share Rate** | (WVI shares) / (Total active users) | Social analytics |
| **Partner Ratio** | (Active partners) / (Total users) | Partner dashboard |

## 7. Сценарии масштабирования

### Когда K-factor становится критическим

При базе **10,000 users и K = 0.6:**

```
Месяц 0: 10,000 users
Месяц 1: +6,000 (от K) + 500 (paid) - 800 (churn) = 15,700
Месяц 2: +9,420 + 500 - 1,256 = 24,364
Месяц 3: +14,618 + 500 - 1,949 = 37,533
```

→ **С 10K до 37K за 3 месяца без роста paid бюджета.**

### Что нужно для достижения K = 0.8

| Действие | Impact на K |
|----------|:-----------:|
| Улучшить WVI share card (CTR 1.2% → 2.0%) | +0.08 |
| Добавить «Wellex Wrapped» (декабрь) | +0.15 (seasonal) |
| Запустить Country Captain program | +0.05 |
| Увеличить partner ratio с 12% до 18% | +0.09 |
| B2B Phase 2 запуск (5 клиентов/мес) | +0.04 |
| **Итого дополнительно** | **+0.41** |

*K 0.38 + 0.41 = K 0.79 — практически достигаем target K = 0.8*

## 8. Культура и операционная модель

### Принципы

| # | Принцип | Как реализуется |
|---|---------|----------------|
| 1 | **Remote-first** | Нет офиса. Все работают удалённо. Timezone overlap: 4ч/день (UTC+2 to UTC+8) |
| 2 | **Async by default** | Документы > звонки. Loom > meetings. Все решения записываются |
| 3 | **Transparency** | Все KPI публичны внутри команды. OKR квартально. Weekly all-hands |
| 4 | **Ship fast, fix fast** | 2-недельные спринты. MVP > perfect. Hotfix < 4ч |
| 5 | **Security-first** | Multisig на всё. Code review обязателен. No single point of failure |

### Инструменты

| Категория | Инструмент |
|-----------|-----------|
| Communication | Slack / Telegram (internal) |
| Project Management | Linear |
| Documentation | Notion |
| Code | GitHub (private repos) |
| Design | Figma |
| CI/CD | GitHub Actions + Vercel |
| Monitoring | Grafana + Forta + PagerDuty |

## 10. Key Person Risk Mitigation

| Мера | Описание | Срок |
|------|----------|:----:|
| **CTO как сооснователь** | Полноценный технический partner, не наёмный сотрудник | M1 |
| **Multisig 3-of-5** | CEO + CTO + Head of DeFi + Legal Advisor + Board Advisor | M2 |
| **Key Person Insurance** | $5M полис на CEO | M3 |
| **Операционное документирование** | Каждое стратегическое решение → Notion с обоснованием | Ongoing |
| **Bus factor ≥ 2** | Каждая критическая система имеет минимум 2 человека с доступом | M4 |
| **Succession planning** | CTO = acting CEO при unavailability. Board может назначить interim CEO | M6 |

## 2.2 Модуль: Кошелёк (Wallet)

### 2.2.1 Обзор баланса

| Поле | Описание |
|------|----------|
| Общий баланс (USD) | Суммарный: депозит + yield + реинвест |
| Основной депозит | Средства в WellexVault |
| Накопленный yield | Сумма всех начисленных доходов |
| Партнёрские бонусы | Сумма всех MLM-выплат |
| Доступно к выводу | После истечения lock-up периодов |
| Заблокировано | В lock-up (с датой разблокировки) |

### 2.2.2 Мультисетевой кошелёк

Отображение балансов по сетям:
- Ethereum, Polygon, BSC, Arbitrum, Optimism, Base, Avalanche
- Единый агрегированный баланс
- Переключатель: «Все сети» / конкретная сеть
- Адреса кошельков (с кнопкой копировать для каждой сети)

### 2.2.3 Пополнение (Crypto)

**Поддерживаемые токены:** USDC, USDT
**Поддерживаемые сети:** ETH, Polygon, BSC, Arbitrum, Optimism, Base, Avalanche

UI Flow:
1. Выбор токена (USDC / USDT)
2. Выбор сети
3. Отображение адреса для пополнения + QR-код
4. Минимальная сумма: $10
5. Подтверждение: «Ваши средства поступят после X подтверждений сети»
6. Статус поступления в real-time

**Поля для хранения:**
- `deposit_address` (адрес кошелька пользователя по каждой сети)
- `tx_hash` входящей транзакции
- `amount_usdc`
- `network`
- `confirmations`
- `status`: pending / confirmed / failed

### 2.2.4 Пополнение (Карта — Card On-Ramp)

Провайдеры: MoonPay / Transak (автовыбор по геолокации)
Методы: Visa / Mastercard / Apple Pay / Google Pay

```
┌─────────────────────────────────────────────┐
│  💳 Пополнение карты                        │
│                                             │
│  Сумма: [$500]  [USD ▾]                     │
│                                             │
│  Способ оплаты:                             │
│  ● Visa / Mastercard                        │
│  ○ Apple Pay                                │
│  ○ Google Pay                               │
│                                             │
│  Провайдер: MoonPay (рекомендован для UAE)  │
│  Конвертация: USD → USDC (Polygon)          │
│  Вы получите: ≈ $487.50 USDC               │
│  Комиссия провайдера: ~2.5%                 │
│                                             │
│  [Продолжить →]                             │
└─────────────────────────────────────────────┘
```

**Важно:** бонусы партнёрской программы считаются от суммы ПОСЛЕ комиссии on-ramp провайдера.

### 2.2.5 Вывод средств

**Требования:**
- KYC верифицирован
- Lock-up период истёк (для конкретного депозита)
- Сумма ≥ $10
- Withdraw approval от администратора (если сумма > $5,000)

**UI Flow:**
1. Выбор суммы (или «Максимум»)
2. Ввод адреса кошелька для вывода
3. Выбор сети
4. Предварительный расчёт: сумма к получению — gas fee
5. Подтверждение (2FA обязательно)
6. Статус: pending → broadcasting → confirmed

**Поля вывода:**
- `withdrawal_address` (куда выводить)
- `amount_usdc`
- `network`
- `gas_fee`
- `net_amount` (итого к получению)
- `status`: pending_approval / approved / broadcasting / confirmed / failed
- `tx_hash` исходящей транзакции

### 2.2.6 История транзакций

**Типы транзакций:**
- DEPOSIT (пополнение)
- WITHDRAWAL (вывод)
- YIELD (начисление дохода)
- REINVEST (реинвестирование)
- PARTNER_BONUS (партнёрский бонус)
- RANK_BONUS (бонус за ранг)
- SUBSCRIPTION_FEE (списание подписки $19/мес)

**Таблица транзакций:**

| Поле | Тип | Описание |
|------|-----|----------|
| date | timestamp | Дата и время (timezone пользователя) |
| type | enum | Тип транзакции |
| amount_usdc | decimal | Сумма в USDC |
| network | string | Сеть (Polygon, ETH, etc.) |
| tx_hash | string | Хэш транзакции (ссылка на explorer) |
| fee_usdc | decimal | Комиссия сети |
| wvi_at_time | int | WVI в момент транзакции |
| yield_rate | decimal | Применённая ставка доходности |
| status | enum | confirmed / pending / failed |
| method | string | Метод пополнения (crypto / card / MoonPay / Transak) |

**Фильтры:**
- По типу транзакции
- По дате (от/до)
- По сети
- По статусу
- По сумме (от/до)

**Экспорт:** CSV / PDF (налоговый отчёт с QR-верификацией) / JSON

## 2.4-BIS Модуль: Развитие партнёрской сети (Network Growth Center)

> Центральный инструмент для активного роста. Всё, что нужно партнёру, чтобы строить сеть осознанно, а не вслепую.

### 2.4-BIS.1 Концепция

Отдельный раздел (или вкладка внутри модуля «Партнёрская сеть») — **персональный штаб по развитию сети**.
Партнёр заходит и сразу видит: что работает, что не работает, где узкое место, кого реактивировать, как удвоить доход за 30 дней.

AI-ассистент здесь — не чат-бот, а **стратег**: анализирует данные, ставит конкретные задачи, даёт инструменты для каждого шага.

### 3.4.2 Управление выводами (Withdrawal Approvals)

**Workflow:**

```
Пользователь запросил вывод
        ↓
Авто-проверка:
  • KYC уровень соответствует сумме?
  • Lock-up истёк?
  • Fraud Score < 70?
  • Не в списке заморозки?
        ↓
Если сумма < $5,000 AND fraud score < 30:
  → Авто-апрув → Broadcast to blockchain
        ↓
Если сумма ≥ $5,000 OR fraud score ≥ 30:
  → Manual Review (Admin)
        ↓
Admin видит в очереди:
  [User: @sofia_r | $7,800 | Polygon | KYC2 | Fraud: 12]
  [Апрувить] [Отклонить с причиной] [Запросить документы]
        ↓
При апруве > $10,000: требуется второй Admin (dual approval)
```

**Очередь ожидающих выводов:**
- Отсортировано по времени ожидания (FIFO)
- Фильтр: по сумме / по fraud score / по KYC уровню
- Bulk actions: апрувить всё до $5,000 с fraud < 20

**SLA:** все выводы должны быть обработаны в течение 4 часов (рабочее время)

## 3.10 Безопасность и Compliance

### 3.10.1 KYC-очередь

**Список заявок на верификацию:**
- User ID / Имя / Страна / Уровень запрошен / Дата подачи / Статус
- Документы (только для Compliance Officer): просмотр скана
- Действия: Апрувить KYC1 / Апрувить KYC2 / Отклонить (с причиной) / Запросить доп. документы
- SLA: обработка в течение 24 часов
- Интеграция с Sumsub (авто-верификация первой линии)

### 3.10.2 AML / Fraud Monitoring

**Правила автоматической пометки (configurable):**
- Вывод > $10,000 за 24ч впервые
- Множественные попытки вывода с новых адресов
- Круговые транзакции (A → B → A)
- Регистрация с IP-адреса из санкционного списка
- Несоответствие геолокации (регистрация Dubai, транзакции Russia + санкции)
- Депозит + немедленный вывод без yield (wash pattern)

**Fraud Score (ML-модель 0–100):**
- 0–30: Low Risk (авто-апрув)
- 30–70: Medium Risk (Manual Review)
- 70–100: High Risk (заморозка + ручная проверка)

**Интерфейс Fraud Alerts:**
- Список всех активных алертов
- Детали: User ID / Тип алерта / Сумма / Дата / IP / Device
- Действия: Dismiss (false positive) / Block / Escalate to Compliance

### 3.10.3 Sanctions Screening

- Автоматическая проверка по OFAC / EU / UN sanctions lists
- Проверка при регистрации + при каждом выводе > $1,000
- Алерт если совпадение → автозаморозка → ручная проверка

### 3.10.4 Audit Log

**Полный журнал всех Admin-действий:**

| Поле | Описание |
|------|----------|
| Timestamp | UTC |
| Admin ID | Кто сделал |
| Action Type | Тип действия |
| Target User ID | На кого направлено |
| Before / After | Состояние до и после |
| Comment | Обязательный комментарий |
| IP Address | IP Admin'а |

**Неизменяемый лог:** записи нельзя удалить или изменить
**Экспорт:** CSV для внешнего аудита

### 3.10.5 Соответствие (Regulatory)

- GDPR запросы (Delete / Export данных пользователя)
- Income Disclosure Statement (ежемесячное обновление)
- FTC 70% Rule мониторинг (см. 3.6.4)
